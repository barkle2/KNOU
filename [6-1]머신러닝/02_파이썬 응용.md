## 2강 파이썬 응용

### 1. 핵심 요약 (Executive Summary)

* 메모리 효율적 데이터 처리: 반복자(Iterator)와 제너레이터(Generator)를 통해 대용량 데이터를 메모리에 한꺼번에 올리지 않고 하나씩 처리함으로써 효율성을 극대화할 수 있습니다.
* 코드의 모듈화와 재사용성: 함수의 정의와 매개변수(Parameter) 활용은 코드 중복을 방지하고 유지보수 및 디버깅을 용이하게 합니다. 특히 가변 매개변수(*args, **kwargs)는 유연한 인터페이스 설계를 가능하게 합니다.
* 고급 함수 기술: 람다(Lambda), 클로저(Closure), 데코레이터(Decorator) 등 파이썬 특유의 함수 응용 기술은 함수를 데이터처럼 다루는 고차 프로그래밍을 지원합니다.
* 객체지향 설계(OOP): 클래스를 설계도로 삼아 속성(Attribute)과 메서드(Method)를 갖춘 인스턴스(Instance)를 생성함으로써 복잡한 시스템을 직관적으로 모델링할 수 있습니다.

### 2. 반복자(Iterator)와 데이터 처리

반복자는 데이터를 하나씩 추출하여 처리하는 메커니즘으로, 특히 대규모 데이터셋을 다룰 때 유용합니다.

#### 2.1. 반복 가능 객체(Iterable)와 반복자(Iterator)

* 반복 가능 객체(Iterable): 요소를 하나씩 추출할 수 있는 객체입니다. 리스트, 튜플, 문자열, 딕셔너리 등이 이에 해당합니다.
* 반복자(Iterator): 반복 가능 객체에서 요소를 실제로 하나씩 꺼내기 위한 객체입니다. iter() 함수로 생성하며, next() 함수를 통해 다음 요소를 호출합니다.
* 메모리 효율성: 10GB와 같은 대용량 리스트를 처리할 때 전체를 메모리에 로드하는 대신, 이터레이터를 사용하여 한 번에 하나씩 처리하면 메모리 사용량을 획기적으로 줄일 수 있습니다.

#### 2.2. 동작 특성

* 단방향성: next()로 요소를 한 번 호출하면 다시 이전으로 돌아가거나 재호출할 수 없습니다.
* StopIteration: 모든 요소를 다 인출한 후 다시 호출하면 StopIteration 오류가 발생하며 반복이 종료됩니다.

### 3. 함수의 구조와 고급 활용

함수는 특정 작업을 수행하는 코드의 모음이며, 프로그램의 구조를 개선하는 핵심 단위입니다.

#### 3.1. 기본 구조 및 용어 구분

* 매개변수(Parameter) vs 인수(Argument): 매개변수는 함수 정의 시 사용하는 변수명이며, 인수는 함수 호출 시 실제로 전달하는 값입니다.
* 반환값(Return Value): return 키워드는 결과를 반환하고 함수를 즉시 종료합니다. 반환값이 명시되지 않은 함수는 None을 반환합니다.

#### 3.2. 매개변수 활용 기법

|구분|설명|사용 예시|
|:-:|:-:|:-:|
|기본 인수(Default Argument)|매개변수에 미리 기본값을 지정하여 호출 시 생략 가능하게 함|def greet(name, msg="안녕하세요")|
|위치 인수(Positional Argument)|입력되는 순서에 따라 매개변수에 할당|greet("민수", "반가워")|
|가변 위치 매개변수(*args)|인수의 개수를 제한하지 않고 튜플 형태로 받음|def add_all(*args)|
|가변 키워드 매개변수(**kwargs)|키와 값의 쌍으로 인수를 받으며 딕셔너리 형태로 처리|def intro(**kwargs)|

#### 3.3. 변수의 범위(Scope)

* 지역 변수(Local Variable): 함수 내부에서만 유효하며 종료 시 소멸합니다.
* 전역 변수(Global Variable): 프로그램 전체에서 접근 가능합니다. 함수 내에서 전역 변수를 수정하려면 global 키워드를 사용해야 하지만, 가급적 권장되지 않습니다.
* 논로컬(Nonlocal): 중첩 함수에서 상위 함수의 변수를 수정할 때 사용합니다.

### 4. 다양한 함수 응용 패턴

파이썬은 함수를 일급 객체로 취급하여 다양한 고차 함수 기능을 제공합니다.

* 람다 함수(Lambda): 한 줄로 정의되는 익명 함수로, 간단한 연산이 필요할 때 사용합니다.
* 재귀 함수(Recursive Function): 자기 자신을 호출하는 함수로, 팩토리얼이나 피보나치 수열 구현에 적합합니다.
* 중첩 함수 및 클로저(Closure): 함수 내부에서 정의된 내부 함수가 외부 함수의 변수를 기억하고 유지하는 방식입니다.
* 고차 함수(Higher-order Function): 함수를 인수로 받거나 함수를 결과로 반환하는 함수입니다.
* 데코레이터(Decorator): 기존 함수의 코드를 수정하지 않고 전후에 기능을 추가할 때 사용하며, @데코레이터명 형식을 취합니다.
* 제너레이터(Generator): yield 키워드를 사용하여 이터레이터를 생성하는 함수입니다. 실행 상태를 유지하며 값을 하나씩 내보냅니다.

### 5. 클래스(Class)와 객체지향 프로그래밍

클래스는 데이터(속성)와 기능(메서드)을 결합하여 실제 사물을 소프트웨어로 모델링하는 설계도입니다.

#### 5.1. 주요 구성 요소

* 속성(Attribute): 인스턴스가 가지는 고유한 데이터 값입니다.
* 메서드(Method): 인스턴스가 수행하는 동작을 정의한 함수입니다.
* 생성자(__init__): 객체가 생성될 때 자동으로 호출되어 초기 속성을 설정합니다.
* Self: 인스턴스 자기 자신을 가리키는 키워드로, 메서드 내에서 속성에 접근할 때 사용합니다.

#### 5.2. 인스턴스화 및 조작 사례 (자동차 클래스 예시)

제시된 소스 코드 내의 자동차 클래스 분석 결과는 다음과 같습니다.

1. 설계: 색상, 파워, 속도를 속성으로 정의하고 전진(forward), 후진(backward)을 메서드로 정의합니다.
2. 생성: 서로 다른 속성(빨간색/파워1, 파란색/파워2)을 가진 객체들을 독립적으로 생성할 수 있습니다.
3. 상태 변화: 메서드 호출을 통해 내부 속성(현재 속도 등)이 변경되며, 외부에서 직접 속성값을 수정(튜닝)하여 메서드 동작 결과에 영향을 줄 수 있습니다.


--------------------------------------------------------------------------------


### 6. 결론 및 정리

파이썬 응용의 핵심은 효율적인 자원 관리와 코드의 체계적 구조화에 있습니다. 반복자와 제너레이터는 시스템 성능을 최적화하며, 다양한 매개변수 활용과 고급 함수 기법은 유연한 프로그래밍을 가능하게 합니다. 최종적으로 클래스를 통한 객체지향 접근법은 복잡한 데이터와 로직을 관리 가능한 단위로 묶어 소프트웨어의 완성도를 높이는 핵심적인 역할을 수행합니다.
