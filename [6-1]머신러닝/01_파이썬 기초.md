# 제1강 파이썬 기초

본 문서는 한국방송통신대학교 통계·데이터과학과 박준희 교수의 머신러닝 강의를 바탕으로 작성되었습니다. 파이썬의 기초적인 사용법부터 실습 환경 구축, 그리고 핵심 자료 구조와 제어문까지의 내용을 체계적으로 정리하여 학습을 돕고자 합니다.

## 1. 실습 환경 구축 및 설정

머신러닝 학습을 위한 파이썬 환경의 일관성은 매우 중요합니다. 다음의 절차를 통해 실습 환경을 구축할 수 있습니다.

### 1.1 파이썬 설치 및 경로 설정

* 버전: 파이썬 3.8 이상 권장.
* 핵심 주의사항: 설치 파일 실행 시 'Add python.exe to PATH' 항목을 반드시 체크해야 합니다. 이는 어느 경로에서든 파이썬 명령어를 실행할 수 있도록 경로를 추가하는 설정입니다. 만약 이를 생략할 경우 명령 프롬프트에서 파이썬 실행을 위해 전체 경로를 입력해야 하는 번거로움이 발생합니다.

### 1.2 가상 환경(Virtual Environment) 및 프로젝트 폴더

본 강의에서는 모든 작업물을 하나의 공간에서 관리하기 위해 C:\machine_learning이라는 영문 폴더를 생성하여 사용합니다.

* 가상 환경 생성: 명령 프롬프트(CMD)에서 프로젝트 폴더로 이동한 후 python -m venv ML 명령어를 사용합니다. 여기서 'ML'은 가상 환경의 이름입니다.
* 가상 환경 활성화: 생성된 가상 환경 폴더 내의 Scripts\activate 파일을 실행합니다. 활성화 시 프롬프트 앞에 (ML)이 표시됩니다.

### 1.3 필수 패키지 설치

가상 환경이 활성화된 상태에서 pip install 명령어를 통해 다음의 패키지들을 설치합니다.

* 주요 패키지: jupyterlab, numpy, scipy, matplotlib, pandas.
* 확장 기능: 주피터 랩에서 변수 상태를 쉽게 확인하기 위해 lckr-jupyterlab-variable-inspector를 추가로 설치합니다.

## 2. 파이썬 기초 문법 및 자료형

### 2.1 변수와 자료형

변수는 데이터를 저장하는 공간이며, 파이썬에서는 '변수명 = 데이터 값'의 형식으로 할당합니다.

* int (정수): 정수 값을 저장하는 자료형.
* float (실수): 소수점이 포함된 실수 값을 저장하는 자료형.
* str (문자열): 따옴표로 감싸진 문자 데이터. 따옴표 없이 입력하면 변수로 인식되므로 주의가 필요합니다.
* bool (논리형): True 또는 False 값을 가지며, 첫 글자는 반드시 대문자여야 합니다.

### 2.2 연산자와 우선순위

파이썬은 표준 수학 연산 순서인 PEMDAS를 따릅니다.

1. Parentheses (괄호): 최우선 순위.
2. Exponents (지수): 거듭제곱 (**).
3. Multiplication and Division (곱셈과 나눗셈): *, /, % (나머지).
4. Addition and Subtraction (덧셈과 뺄셈): +, -.

* 단항 연산자(부호 표시)는 곱셈/나눗셈보다 우선순위가 높습니다.


--------------------------------------------------------------------------------


## 3. 주요 자료 구조

파이썬에서 여러 데이터를 효율적으로 관리하기 위해 사용되는 세 가지 핵심 구조입니다.

|구분|리스트 (List)|튜플 (Tuple)|딕셔너리 (Dictionary)|
|:-:|:-:|:-:|:-:|
|기호|[ ] (대괄호)|( ) (소괄호)|{ } (중괄호)|
|순서|있음|있음|없음 (Key로 관리)|
|변경 가능성|가능 (Mutable)|불가능 (Immutable)|가능 (Mutable)|
|중복 허용|값 중복 허용|값 중복 허용|Key 중복 불가, Value 중복 허용|
|주요 특징|append, remove 가능|메모리 효율 및 계산 속도 우수|키-값(Key-Value) 쌍으로 저장|

## 4. 제어문 (조건문 및 반복문)

### 4.1 조건문 (if-elif-else)

명제의 참/거짓 여부에 따라 코드 실행을 제어합니다.

* 독립적 사용: 여러 if문을 나열하면 각각의 조건이 서로 영향을 주지 않고 독립적으로 판단됩니다.
* 중첩 사용: 조건문 안에 또 다른 조건문을 넣어 복잡한 알고리즘(예: 대출 가능 여부 판단)을 구현할 수 있습니다.
* 플래그 변수: True나 False 값을 저장하여 상태를 제어하는 변수로 활용됩니다.

### 4.2 반복문 (for, while)

* for 루프: 리스트나 튜플처럼 순회 가능한 객체를 대상으로 반복합니다. range() 함수나 zip() 함수를 사용하여 다수의 리스트를 동시에 순회할 수 있습니다.
* while 루프: 특정 조건이 참인 동안 계속해서 코드를 실행합니다.
* 제어 명령어:
  * break: 반복문의 실행을 완전히 중단하고 탈출합니다.
  * continue: 현재 차례의 실행만 건너뛰고 다음 반복으로 넘어갑니다.

## 6. 퀴즈 정답지

1. 정답: 컴퓨터의 모든 경로에서 파이썬 실행 파일이 없더라도 'python' 명령어를 쳤을 때 실행되도록 경로를 추가하기 위함입니다. 이를 설정하지 않으면 파이썬 실행 시마다 파일의 전체 경로(Full Path)를 입력해야 하는 번거로움이 발생합니다.
2. 정답: 프로젝트별로 독립된 실습 환경을 유지하여 패키지 버전 충돌 등을 방지하기 위해 사용합니다. 활성화 여부는 명령 프롬프트의 경로 앞에 가상 환경 이름(예: (ML))이 표시되는 것을 통해 확인할 수 있습니다.
3. 정답: 명령 프롬프트는 주피터 랩 브라우저 환경이 작동하게 하는 '본체' 역할을 하기 때문입니다. 본체인 명령 프롬프트를 닫으면 실행 중인 주피터 랩 브라우저 서비스도 함께 종료되어 더 이상 사용할 수 없게 됩니다.
4. 정답: 파이썬에서 =은 할당 연산자로, 왼쪽은 변수명, 오른쪽은 데이터 값이 와야 합니다. 따라서 a = 5는 변수 a에 5를 저장하라는 올바른 명령이지만, 5 = a는 상수에 변수를 할당할 수 없으므로 오류가 발생합니다.
5. 정답: 가장 큰 차이는 리스트는 내용 변경이 가능(Mutable)하고 튜플은 불가능(Immutable)하다는 점입니다. 튜플은 데이터가 고정되어 있음을 컴퓨터가 알기 때문에 메모리 사용량이 적고 계산 속도가 빠르다는 장점이 있습니다.
6. 정답: 리스트의 가장 마지막에 위치한 원소를 호출하는 방식입니다. 파이썬에서는 뒤에서부터 순서를 셀 때 -1부터 시작하는 인덱싱을 지원합니다.
7. 정답: 키(Key)는 데이터를 식별하는 고유한 값이어야 하므로 중복이 허용되지 않습니다. 반면 값(Value)은 데이터 자체를 의미하므로 서로 다른 키에 동일한 값이 저장되는 중복이 허용됩니다.
8. 정답: 지수(Exponentiation) 연산자인 **입니다. 파이썬의 표준 수학 연산 순서에 따라 괄호 처리 이후 거듭제곱 연산을 가장 먼저 수행합니다.
9. 정답: 두 개 이상의 리스트를 동시에 순회하며 각각의 원소를 세트로 묶어 처리하고 싶을 때 유용합니다. 예를 들어 이름 리스트와 점수 리스트를 동시에 돌리며 매칭된 정보를 출력할 때 사용됩니다.
10. 정답: break는 조건을 만족하면 반복문 자체를 완전히 종료하고 빠져나오는 명령입니다. 반면 continue는 현재 진행 중인 반복 순서만 건너뛰고 다음 반복 순서를 이어서 수행한다는 점이 다릅니다.

## 8. 주요 용어 사전 (Glossary)

* PATH (경로): 컴퓨터 시스템이 실행 파일을 찾을 때 참조하는 주소 목록입니다.
* 가상 환경 (Virtual Environment): 동일한 시스템 내에서 프로젝트별로 독립된 파이썬 실행 환경을 격리하여 관리하는 기술입니다.
* 루트 디렉토리 (Root Directory): 파일 시스템의 최상위 경로로, 윈도우에서는 보통 C:\를 의미합니다.
* 프롬프트 (Prompt): 명령어를 입력받기 위해 대기하는 표시로, 현재 작업 중인 경로를 알려줍니다.
* 메서드 (Method): 특정 객체(예: 리스트)에 종속되어 기능을 수행하는 함수로, 객체명.메서드명() 형식으로 호출합니다.
* 인덱싱 (Indexing): 데이터 구조 내에서 특정 원소의 위치(번호)를 사용하여 값을 참조하는 행위입니다. 파이썬은 0부터 시작합니다.
* 슬라이싱 (Slicing): 데이터 구조의 일부분을 잘라내어 부분 집합을 만드는 기법입니다.
* 뮤테이블 (Mutable): 생성된 후 그 내용을 수정할 수 있는 객체의 속성입니다 (예: 리스트).
* 이뮤테이블 (Immutable): 생성된 후 그 내용을 절대 변경할 수 없는 객체의 속성입니다 (예: 튜플).
* 명제 (Proposition): 참과 거짓을 명확하게 판별할 수 있는 문장입니다.
* 플래그 변수 (Flag Variable): 프로그램의 흐름을 제어하거나 특정 상태를 기록하기 위해 사용하는 논리형(Boolean) 변수입니다.
* PEMDAS: 연산 우선순위 규칙 (Parentheses, Exponents, Multiplication/Division, Addition/Subtraction).
* 마크다운 (Markdown): 주피터 랩 등에서 서식이 있는 문서를 작성하기 위해 사용하는 경량 마크업 언어입니다.
* 커널 (Kernel): 주피터 랩에서 사용자가 작성한 코드를 실제로 실행하고 결과를 반환하는 엔진입니다.
